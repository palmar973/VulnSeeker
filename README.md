## рҹ”Қ ВҝQuГ© es VulnSeeker?

**VulnSeeker** es un framework DAST (Dynamic Application Security Testing) avanzado, desarrollado en Python, diseГұado para realizar auditorГӯas de seguridad automatizadas de forma integral. A diferencia de escГЎneres bГЎsicos que se limitan a un solo vector de ataque, VulnSeeker orquesta **25 mГіdulos independientes** de anГЎlisis вҖ” desde inyecciones SQL y XSS hasta reconocimiento OSINT de subdominios, consulta de CVEs en tiempo real contra la NVD del NIST, y anГЎlisis con Inteligencia Artificial.

El sistema opera bajo una arquitectura modular extensible donde cada mГіdulo hereda de la clase base `ScannerModule` y se registra en el motor central, permitiendo agregar nuevas capacidades de escaneo sin modificar el nГәcleo del sistema.

### ВҝPor quГ© VulnSeeker?

- **Cobertura integrada.** VulnSeeker ejecuta una cadena completa: reconocimiento вҶ’ fingerprinting вҶ’ crawling вҶ’ descubrimiento API-aware вҶ’ ataque multihilo вҶ’ deduplicaciГіn вҶ’ anГЎlisis IA вҶ’ reporte.
- **Descubrimiento API-aware (SPAs).** Cuando el objetivo es una Single-Page Application (Angular/React/Vue) cuyo crawling estГЎtico no revela endpoints, VulnSeeker recupera la superficie de API parseando la especificaciГіn OpenAPI/Swagger y extrayendo rutas `/api/` y `/rest/` de los *bundles* JavaScript вҖ”sin navegador *headless*вҖ” y ataca tambiГ©n cuerpos JSON.
- **Robustez ante falsos positivos.** Detecta respuestas *catch-all* (soft-404) para no reportar archivos inexistentes en servidores que responden 200 a todo (tГӯpico de SPAs), y valida contra *baseline* en los mГіdulos de inyecciГіn.
- **8/10 categorГӯas del OWASP Top 10 (2021).** Las 2 restantes (A08, A09) no son evaluables externamente mediante DAST.
- **Inteligencia Artificial integrada.** Llama 3.3 70B realiza un anГЎlisis contextual de los hallazgos tГ©cnicos y los prioriza segГәn el riesgo de negocio.
- **CVEs en tiempo real.** IntegraciГіn con la API 2.0 del NIST NVD para consultar vulnerabilidades conocidas asociadas a las tecnologГӯas detectadas.
- **194 tests unitarios.** Suite completa con `pytest` y pipeline CI/CD en GitHub Actions.
- **Validado contra el OWASP Benchmark.** MediciГіn cuantitativa con *ground truth* etiquetado: ГҚndice de Youden 0.68 sobre el subconjunto atacable (SQLi 0.89, XSS 0.87, PrecisiГіn 0.95), reportado con honestidad metodolГіgica.
- **Persistencia histГіrica.** Cada escaneo se almacena en SQLite, permitiendo anГЎlisis de tendencias y comparaciГіn entre auditorГӯas.

## вҡЎ MГіdulos de DetecciГіn

### InyecciГіn y EjecuciГіn (A03 + A10 вҖ” Injection / SSRF)

| MГіdulo | DescripciГіn | Severidad |
|--------|-------------|-----------|
| **SQL Injection** | SQLi basada en errores (MySQL, PostgreSQL, Oracle, MSSQL, SQLite/ORM), ciega *time-based* e inyecciГіn en cuerpos JSON de APIs REST | рҹ”ҙ CRITICAL |
| **Cross-Site Scripting** | IdentificaciГіn de XSS reflejado mediante inyecciГіn de canarios | рҹҹ  HIGH |
| **Command Injection** | DetecciГіn de ejecuciГіn remota de comandos (RCE) en parГЎmetros | рҹ”ҙ CRITICAL |
| **Local File Inclusion** | Pruebas de traversal de directorios para lectura de archivos locales | рҹ”ҙ HIGH |
| **Remote File Inclusion** | DetecciГіn de inclusiГіn de archivos remotos en aplicaciones web | рҹ”ҙ HIGH |
| **SSRF Scanner** | DetecciГіn heurГӯstica in-band de Server-Side Request Forgery | рҹ”ҙ HIGH |

### Control de Acceso y AutenticaciГіn (A01 + A07)

| MГіdulo | DescripciГіn | Severidad |
|--------|-------------|-----------|
| **Open Redirect** | DetecciГіn de redirecciones abiertas explotables | рҹҹ  HIGH |
| **CORS Scanner** | AnГЎlisis de polГӯticas Cross-Origin mal configuradas | рҹҹЎ MEDIUM |
| **CSRF Auditor** | VerificaciГіn de protecciГіn anti-falsificaciГіn en formularios | рҹҹЎ MEDIUM |
| **Brute Force Detector** | DetecciГіn de endpoints sin protecciГіn contra fuerza bruta | рҹҹ  HIGH |
| **Weak Session Auditor** | AuditorГӯa de gestiГіn de sesiones (entropГӯa, rotaciГіn, tokens) | рҹҹ  HIGH |

### ConfiguraciГіn y Hardening (A05 вҖ” Security Misconfiguration)

| MГіdulo | DescripciГіn | Severidad |
|--------|-------------|-----------|
| **Header Analyzer** | EvaluaciГіn de cabeceras de seguridad (CSP, HSTS, X-Frame-Options, etc.) | рҹҹЎ MEDIUM |
| **Dir Listing Detector** | DetecciГіn de listado de directorios habilitado | рҹҹЎ MEDIUM |
| **HTTP Method Scanner** | IdentificaciГіn de mГ©todos HTTP peligrosos habilitados (PUT, DELETE, TRACE) | рҹҹЎ MEDIUM |
| **Exposure Scanner** | DetecciГіn de archivos sensibles expuestos (.env, .git, backups) | рҹҹ  HIGH |
| **Path Fuzzer** | Descubrimiento de rutas y archivos ocultos mediante diccionarios | рҹҹЎ MEDIUM |

### Componentes Vulnerables y CriptografГӯa (A02 + A06)

| MГіdulo | DescripciГіn | Severidad |
|--------|-------------|-----------|
| **CVE Lookup** | Base de datos local + consulta en vivo a NVD API 2.0 del NIST | рҹ”ҙ CRITICAL |
| **TLS Checker** | ValidaciГіn de certificados SSL/TLS, protocolos y cifrados dГ©biles | рҹҹ  HIGH |
| **Sensitive Data Scanner** | DetecciГіn de datos sensibles expuestos (emails, tokens, API keys) | рҹҹ  HIGH |

### DiseГұo Inseguro (A04)

| MГіdulo | DescripciГіn | Severidad |
|--------|-------------|-----------|
| **File Upload Detector** | DetecciГіn de formularios de subida de archivos sin validaciГіn | рҹҹ  HIGH |

### Reconocimiento y AnГЎlisis

| MГіdulo | DescripciГіn |
|--------|-------------|
| **Web Crawler** | Explorador estructural que mapea URLs, formularios y puntos de entrada con soporte de autenticaciГіn |
| **API Endpoint Discovery** | Recupera endpoints de API REST en SPAs sin navegador: parseo de OpenAPI/Swagger + extracciГіn de rutas en *bundles* JavaScript |
| **Tech Fingerprinter** | IdentificaciГіn de servidor, lenguaje backend, CMS/Framework y versiones |
| **Subdomain Scanner** | Descubrimiento pasivo OSINT vГӯa crt.sh y HackerTarget con validaciГіn de subdominios live |
| **Port Scanner** | Escaneo de puertos TCP abiertos con identificaciГіn de servicios |

### Inteligencia Artificial

| CaracterГӯstica | Detalle |

| **Output** | Nivel de riesgo global, top amenazas, plan de acciГіn inmediato (48h) y recomendaciones estratГ©gicas |

## вҡ пёҸ Limitaciones Conocidas

- **DetecciГіn heurГӯstica:** Los mГіdulos de inyecciГіn combinan tГ©cnicas *error-based*, detecciГіn ciega *time-based* (SQLi y command injection), inyecciГіn en cuerpos JSON y un componente compartido de vectores (`injection_points`). No se detectan aГәn XSS almacenado (*stored*) ni DOM XSS, que requieren seguimiento de estado o ejecuciГіn de JavaScript del lado cliente.
- **SSRF in-band:** DetecciГіn por anГЎlisis de respuesta HTTP; no usa callbacks OOB (Out-of-Band).
- **Subdominios como recon:** Los subdominios descubiertos vГӯa OSINT no se escanean automГЎticamente (requiere autorizaciГіn explГӯcita).
- **IA dependiente de API:** El informe ejecutivo requiere conectividad con Groq API; el resto del framework opera offline.
- **SPAs sin spec ni rutas legibles:** El descubrimiento API-aware cubre SPAs que publican OpenAPI/Swagger o referencian sus rutas en el JavaScript; aquellas que ofusquen por completo sus endpoints requerirГӯan un motor de renderizado *headless* (trabajo futuro).
- **ValidaciГіn en entorno controlado:** 0 falsos positivos observados contra DVWA (low security), con validaciГіn adicional contra Altoro Mutual y la SPA OWASP Juice Shop; en producciГіn los resultados pueden variar.
- **LГӯmites cuantificados contra el OWASP Benchmark:** la mediciГіn con *ground truth* revelГі que VulnSeeker es ciego ante el *path traversal* y el *command injection* sin reflejo, y que no cubre vectores de inyecciГіn por cabecera o cuerpo XML (вүҲ34% de los casos). Se incorporaron tГ©cnicas ciegas *time-based* (cmdi) y *error-based* (LFI) para esos casos, pero se verificГі empГӯricamente que el banco sintГ©tico **no las expone**: no bloquea la respuesta con el retardo del comando (`sleep 10` вҶ’ ~0.06 s) ni propaga la traza de error de archivo. Su eficacia se materializa en apps reales (DVWA), no en el orГЎculo. La carencia de XSS por `POST` que la mediciГіn destapГі sГӯ fue subsanada extendiendo el mГіdulo (Youden XSS 0.38 вҶ’ 0.87). Son lГӯneas de mejora abordables sin tocar la arquitectura modular.

Para mГЎs detalles, consulte la secciГіn de Limitaciones del Estudio en el [documento de tesis](tesis/main.pdf).
